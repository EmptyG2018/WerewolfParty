# 需求同步指令

当用户的提示词以「【需求】」为开头时，执行以下操作：

## 步骤

1. **提取需求内容**：去掉「【需求】」前缀，提取剩余文本作为需求内容
2. **读取 CLAUDE.md**：读取当前项目根目录下的 `CLAUDE.md` 文件
3. **判断需求类型**：根据内容判断是「新增需求」还是「变更需求」
4. **追加到 CLAUDE.md**：在文档末尾的「需求变更记录」章节追加记录，格式如下：

```markdown
## 需求变更记录

### [日期] 需求标题
- **类型**: 新增/变更/移除
- **内容**: 需求描述
- **影响范围**: 涉及的模块或功能
```

5. **同步代码变更**：如果需求涉及功能变更，同时更新 CLAUDE.md 中对应章节（如数据结构、Socket 事件、游戏规则等）
6. **确认完成**：输出同步结果摘要

## 规则

- 如果 CLAUDE.md 末尾没有「需求变更记录」章节，自动创建
- 每条需求记录包含日期（自动获取当前日期）
- 保持原有文档结构不变，只追加或更新相关章节
- 如果需求与现有内容冲突，以新需求为准并标注变更

$ARGUMENTS
